VIRTUAL MACHINE

VM Architecture

Primary VM (Layer 4a)

Full Lua instruction set with 34 opcodes: LOADK, MOVE, ADD, SUB, MUL, DIV, CALL, RETURN, FORLOOP, CLOSURE, GETTABLE, SETTABLE, SELF, and more. Opcode table shuffled per build.

Mini VM (Layer 4b)

Secondary lightweight VM wraps the primary VM output. Encrypted with separate key arrays. The VM runtime itself is hidden inside an encrypted payload decoded at runtime.

Encrypted Constant Pool

All constants — strings and numbers — stored in an encrypted pool decoded at runtime. Two decryption functions (XOR + rotation) are randomly assigned per constant.

Polymorphic Builds

New random seed each run → different variable names, XOR keys, opcode mappings, dead code patterns. Two obfuscations of identical source produce completely different output.
